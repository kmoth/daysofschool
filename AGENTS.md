# AGENTS.md

## Project Role

Act as a Senior Full-Stack Engineer for this project. This is a small Vite + React static web app that counts school days and school time remaining from a local configuration file. Keep changes simple, user-facing behavior clear, and calendar/countdown logic well tested.

## Tech Stack

- Runtime/build: Vite 7 with `@vitejs/plugin-react`
- UI: React 19 with JSX
- Language: plain JavaScript ES modules
- Styling: global CSS in `styles.css`
- Tests: Node.js built-in test runner (`node --test`)
- Linting: ESLint flat config in `eslint.config.js`
- Deployable artifact: generated `dist/` directory from `npm run build`

## Setup & Commands

- Install dependencies: `npm install`
- Start dev server: `npm run dev`
- Before starting a dev server, check whether one is already running at `http://127.0.0.1:5173`; the project owner typically keeps Vite running there.
- Lint: `npm run lint`
- Test: `npm test`
- Build: `npm run build`
- Full local check: `npm run check`
- Preview production build: `npm run preview`

## Coding Standards

- Follow the existing plain JavaScript + JSX style. Do not introduce TypeScript unless the project is explicitly migrated.
- Keep calendar and countdown logic pure where possible, with React components responsible for rendering and interaction.
- Use local date construction and the helpers in `src/schedule.js` for date parsing/formatting; avoid timezone-sensitive `Date` string parsing in new logic.
- Add or update tests in `test/` when changing countdown math, schedule normalization, date handling, or focus behavior.
- Keep UI state close to the component that owns it. Put reusable hooks in `src/hooks/` and reusable visual pieces in `src/components/`.
- Respect the ESLint rules already configured: strict equality, no duplicate imports, no unused variables, and no console output unless intentional.
- Keep copy concise and child-friendly, since this app is a visual countdown for school days.

## Boundaries

### Never

- Never add new dependencies without asking first.
- Never edit `dist/` by hand; it is generated by `npm run build`.
- Never replace the existing Vite/React setup with another framework without explicit approval.
- Never change `school-config.js` dates, holidays, or school hours unless the task is specifically about calendar configuration.
- Never use destructive git commands or revert user changes unless explicitly asked.
- Never weaken validation or tests around school days, days off, school hours, or date parsing.

### Always

- Always inspect existing patterns before changing structure or adding abstractions.
- Always preserve user edits in the working tree and keep unrelated files untouched.
- Always run the smallest relevant verification command after a change, and prefer `npm run check` before handing off substantial code changes.
- Always update `README.md` if setup, deployment, or public commands change.
- Always keep deploy guidance centered on the built `dist/` directory, not the repository root.

## Context & Directory Structure

- `index.html`: Vite entry HTML for local development.
- `src/main.jsx`: React entry point.
- `src/App.jsx`: Main app shell, mode selection, countdown state, palette selection, and page composition.
- `src/countdown.js`: Countdown-specific calculations, headline text, summer countdown helpers, and date range handling.
- `src/schedule.js`: Schedule normalization, date parsing, school-day checks, remaining-time calculations, and day-off formatting.
- `src/calendarMath.js`: Calendar layout/math utilities.
- `src/focus.js`: Focus state for selected calendar/highlight days.
- `src/devTime.js`: Query-string preview/test time helpers.
- `src/ui.js`: Small UI utility helpers.
- `src/palettes.js`: Palette definitions and preference helpers.
- `src/components/`: React components for the header, highlights, calendar, summer mode, confetti, palette switcher, and test page.
- `src/hooks/`: Reusable React hooks for time updates, layout CSS variables, attention pulse, calendar focus, standalone display mode, and scrolling.
- `school-config.js`: Current school year, summer break, school hours, school weekdays, and configured days off.
- `styles.css`: Global app styling and component classes.
- `test/`: Node test files for countdown and schedule behavior.
- `public/site.webmanifest`: Static web app manifest.
- `vite.config.js`: Vite config, including relative `base: "./"` for static deployment.
- `eslint.config.js`: ESLint flat config.
- `package.json`: Scripts and dependencies.
- `README.md`: Human-facing setup, build, deploy, and feature notes.
